## Design Philosophy

The SmartCrab CLI provides a Rails-like developer experience. Developers work within the framework, and code generators automate the creation of boilerplate code.

### Comparison with Rails

| Rails | SmartCrab | Description |
|-------|-----------|------|
| `rails new` | `crab new` | Project generation |
| `rails generate model` | `crab generate dto` | Data structure generation |
| `rails generate controller` | `crab generate layer` | Processing unit generation |
| N/A | `crab generate graph` | Graph definition generation |
| `rails server` | `crab run` | Application execution |

## Command Structure

```
crab
├── new <project-name>       # Generate a new project
├── generate (g)             # Code generators
│   ├── node <name>          # Generate a Node
│   ├── dto <name>           # Generate a DTO
│   └── graph <name>         # Generate a Graph definition
├── viz [graph]              # Visualize Graph definitions
└── run                      # Run the application
```

### CLI Framework

Uses the `clap` crate.

## Project Generation Template Design

The project structure generated by `crab new <project-name>`:

```
<project-name>/
├── Cargo.toml
├── SmartCrab.toml           # SmartCrab configuration file
├── Dockerfile
├── compose.yml              # Development services (Jaeger, etc.)
├── src/
│   ├── main.rs              # Entry point (Runtime startup)
│   ├── dto/
│   │   └── mod.rs
│   ├── layer/
│   │   ├── mod.rs
│   │   ├── input/
│   │   │   └── mod.rs
│   │   ├── hidden/
│   │   │   └── mod.rs
│   │   └── output/
│   │       └── mod.rs
│   └── graph/
│       └── mod.rs
└── tests/
    └── integration/
        └── mod.rs
```

### Generated `main.rs`

```rust
use smartcrab::prelude::*;
use smartcrab::runtime::Runtime;

mod dto;
mod node;
mod graph;

#[tokio::main]
async fn main() -> Result<()> {
    smartcrab::telemetry::init()?;

    Runtime::new()
        // .add_graph(graph::my_graph()?)
        .run()
        .await
}
```

### Generated `SmartCrab.toml`

```toml
[project]
name = "<project-name>"
version = "0.1.0"

[telemetry]
enabled = true
exporter = "otlp"

[claude_code]
timeout_secs = 300
```

## Node Generator Design

`crab generate node <name>` generates Node boilerplate.

### Generated Targets

| Option | Generated File |
|-----------|-------------|
| `--type input` | `src/node/input/<name>.rs` |
| `--type hidden` | `src/node/hidden/<name>.rs` |
| `--type output` | `src/node/output/<name>.rs` |

### Generation Template Example (Hidden Node)

```rust
// src/node/hidden/{{name}}.rs
use smartcrab::prelude::*;
use crate::dto::{/*{ name | pascal_case }*/Input, /*{ name | pascal_case }*/Output};

pub struct /*{ name | pascal_case }*/ ;

impl Node for /*{ name | pascal_case }*/ {
    fn name(&self) -> &str {
        "/*{ name | pascal_case }*/"
    }
}

#[async_trait]
impl HiddenNode for /*{ name | pascal_case }*/ {
    type Input = /*{ name | pascal_case }*/Input;
    type Output = /*{ name | pascal_case }*/Output;

    async fn run(&self, input: Self::Input) -> Result<Self::Output> {
        todo!("Implement /*{ name | pascal_case }*/ logic")
    }
}
```

### Specifying Input Node Subtypes

```bash
crab generate node webhook_receiver --type input --input-type http
crab generate node daily_check --type input --input-type cron
crab generate node discord_listener --type input --input-type chat
```

`--input-type` generates boilerplate tailored to the specified subtype.

### Specifying Output Node Subtypes

```bash
crab generate node discord_notifier --type output --output-type discord
```

Specifying `--output-type discord` generates a Node that already includes the boilerplate needed for sending to a Discord Webhook (a `webhook_url` field and message sending logic).

## DTO Generator Design

`crab generate dto <name>` generates a DTO struct.

### The `--fields` Option

```bash
crab generate dto analysis_result --fields "severity:String,score:f64,tags:Vec<String>"
```

Generated output:

```rust
use smartcrab::Dto;

#[derive(Dto)]
pub struct AnalysisResult {
    pub severity: String,
    pub score: f64,
    pub tags: Vec<String>,
}
```

If `--fields` is omitted, the struct is generated with empty fields.

## Graph Generator Design

`crab generate graph <name>` generates boilerplate for a Graph definition function.

### Generation Template

```rust
// src/graph/{{name}}.rs
use smartcrab::prelude::*;

pub fn /*{ name | snake_case }*/() -> Result<DirectedGraph> {
    DirectedGraphBuilder::new("/*{ name | snake_case }*/")
        // .add_node(...)
        // .add_edge(...)
        .build()
}
```

## Template Engine

Code generator templates use the following variables for customization.

### Template Variables

| Variable | Description | Example |
|------|------|-----|
| `/*{ name }*/` | The input name as-is | `data_analyzer` |
| `/*{ name \| pascal_case }*/` | Converted to PascalCase | `DataAnalyzer` |
| `/*{ name \| snake_case }*/` | Converted to snake_case | `data_analyzer` |

### Automatic `mod.rs` Updates

The generator not only creates new files but also automatically adds `pub mod` declarations to the corresponding `mod.rs`.

```bash
$ crab generate node data_analyzer --type hidden

Created: src/node/hidden/data_analyzer.rs
Updated: src/node/hidden/mod.rs  (added: pub mod data_analyzer;)
Created: src/dto/data_analyzer.rs  (DataAnalyzerInput, DataAnalyzerOutput)
Updated: src/dto/mod.rs  (added: pub mod data_analyzer;)
```
